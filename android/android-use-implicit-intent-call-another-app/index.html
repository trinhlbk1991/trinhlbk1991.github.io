For example, instead of develop a module to send email, you can call Gmail app to do that for you! This kind of implementation can be found in a lot of application. And it saves you tons of time!

In this post, I‚Äôll show you how to use implicit intent to call another app.

Depend on user action, the data can be very simple like an URI, or complex and have many extra data like reciever, subject, message,‚Ä¶, or have no data required!

If your data is a URI, you can use a simple constructor to create an implicit intent.

  • ¬†Intent to view a web page:

Other kinds of implicit intents require ‚Äúextra‚Äù data that provide different data types, such as a string.

C -¬†Check If There is an App to Recieve the Intent

Although Android OS has some default application to recieve basic Intents such as Phone, Calendar, Email,‚Ä¶ we should¬†always perform¬†a verification step before invoking an intent.

The verification step is quite simple:

	// Verify it resolves
	PackageManager packageManager = getPackageManager();
	List<ResolveInfo> activities = packageManager.queryIntentActivities(intent, 0);
	return activities.size() > 0;

}
